关于自动拆箱的缓存性质研究

在使用中会发现，int与其对应的包装类Integer做 等比较 时，会发现，值在 -128~127 间，可以产生true，但在范围外就会产生预期之外的false，翻看 valueOf() 源码

valueOf()装箱是有缓存机制的，缓存的范围定义在 IntegerCache 类中，low 为固定值-128，high 默认为127，可以通过 sum.misc.VM 这个类，即 JVM 参数 -XX:AutoBoxCacheMax 配置

这个的缓存机制是在static区设置一个数组，容量为128 + IntegerCache.high ，内容为 -128 ~ IntegerCache.high ，被缓存的 Integer 对象中的值引用设置到该数组上，所以做 等比较 时，会产生相等的现象，因为引用相同。

代码分析中已经得出，AutoBoxCacheMax 必须大于127，小于127会自动设置为127。
